# 常见问题

## 连接问题

### Q: 请求超时怎么办？

**A:** 可能的原因和解决方案：

1. **网络延迟过高** — 尝试使用网络质量更好的环境
2. **请求内容过长** — 减少输入 Token 数量
3. **服务端负载高** — 稍后重试，或联系管理员确认服务状态

建议在客户端设置合理的超时时间（30-60 秒）。

### Q: 出现 401 错误怎么排查？

**A:** 按以下步骤排查：

1. 确认 API Key 格式正确（以 `sk-relay-` 开头）
2. 确认 Key 未过期或被禁用
3. 确认 Base URL 配置正确
4. 联系管理员确认 Key 状态

### Q: 连接被拒绝（Connection Refused）？

**A:** 检查以下几点：

1. Base URL 是否正确（注意末尾不要有多余的 `/`）
2. 是否有防火墙或代理拦截
3. 尝试使用 cURL 直接测试连通性

## 计费相关

### Q: 如何查看我的用量？

**A:** 联系管理员获取用量查询页面链接，可以实时查看 Token 消耗明细。

### Q: 计费方式是什么？

**A:** 按实际使用的 Token 数量计费，不同模型有不同的单价。无最低消费限制。

### Q: 有用量上限吗？

**A:** 默认不做用量限制。如果单用户用量影响到其他用户的正常使用，管理员会主动沟通协调。

## 速率限制

### Q: 有请求频率限制吗？

**A:** 有的，为了保证所有用户的服务质量，会根据套餐类型设置不同的速率限制。如需提升限制，请联系管理员。

### Q: 遇到 429 错误怎么办？

**A:** 这是速率限制触发的标志。建议：

1. 降低请求频率
2. 在代码中加入自动重试逻辑
3. 联系管理员了解当前限额

## 兼容性

### Q: 支持哪些客户端？

**A:** 支持所有兼容 OpenAI API 协议的客户端，包括但不限于：

- Cursor
- Claude Code
- Continue (VS Code)
- Cline
- Aider
- OpenAI SDK (Python / Node.js / Go 等)
- 任何自定义的 OpenAI 兼容客户端

### Q: 支持 Function Calling 吗？

**A:** 支持。中转站完全兼容 OpenAI 的 Function Calling / Tool Use 协议。

### Q: 支持流式输出吗？

**A:** 支持。设置 `stream: true` 即可使用流式响应。

## 使用 Tips

### Q: 如何获得最佳体验？

**A:** 几个建议：

1. **选择合适的模型** — 简单任务用 Haiku，日常使用 Sonnet，复杂任务用 Opus
2. **合理控制上下文长度** — 避免发送过多历史消息
3. **使用流式输出** — 长文本生成时使用流式可以更快看到结果
4. **做好错误处理** — 在代码中加入重试机制提升鲁棒性

### Q: 如何降低 Token 消耗？

**A:** 几个实用技巧：

1. 精简 System Prompt，避免冗余描述
2. 控制历史消息轮数，及时清理不相关上下文
3. 简单任务选择 Haiku 等轻量模型
4. 避免在 Prompt 中重复相同内容
